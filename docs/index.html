// what it does

Control your AI agents from anywhere

conductor-telegram bridges your Conductor workspaces to Telegram. Start workspaces, review PRs, and approve agent decisions without opening a laptop.

Launch workspaces remotely

Start new AI agent workspaces from Telegram. Pick a repo, describe the task, and the agent gets to work.

Approve decisions

Agents ask for permission before risky operations. Approve or reject from your phone with a quick reply.

Real-time status

Get progress reports as agents work. See what files changed, what PRs were created, and when things go wrong.

Forum topics per workspace

Each workspace gets its own Telegram forum topic. Clean separation, easy to follow, no noise.

MCP server

Three tools for agents: report_status, report_artifact, and request_human. Agents talk to you through the protocol.

Self-diagnosing

Run doctor to check Node.js, config, bot token, database, Conductor, MCP plugin, and repos. All at once.

Setup in 3 minutes

The setup wizard handles everything. You just need a Telegram bot token.

1

Create a Telegram bot

Open @BotFather in Telegram, send /newbot, and copy the token. That's your BOT_TOKEN.

2

Install and run setup

Install globally with npm, then run the interactive setup wizard. It auto-detects existing .env files.

npm i -g conductor-telegram
conductor-telegram setup
3

Start the bot

Run conductor-telegram to start. Send /setup in your Telegram chat to link the bot. It will detect whether you're in a private chat or a forum group.

conductor-telegram
4

Install the MCP plugin

So agents can talk back to Telegram. This installs the MCP server into Claude Code.

conductor-telegram install-plugin

Configuration

Config is stored at ~/.conductor-telegram/config.json. Precedence: CLI flags > env vars > config.json > defaults.

Architecture

Three components: a CLI that manages config, a Telegram bot that handles user interaction, and an MCP server that agents use to communicate.

src/ bot/ # Telegram bot: commands, middleware, formatting, forum topics cli/ # CLI: setup wizard, doctor, banner, config management mcp/ # MCP server: report_status, report_artifact, request_human store/ # SQLite: database setup and query layer types/ # Shared TypeScript types

Contributing

conductor-telegram is MIT licensed and open to contributions. Here's how to get started.

1

Fork and clone

git clone https://github.com/YOUR_USERNAME/conductor-telegram.git
cd conductor-telegram
npm install
2

Create a branch

git checkout -b feat/your-feature
3

Make your changes

Run npm run dev for live reload during development. Run npm run typecheck before committing.

4

Open a PR

Push your branch and open a pull request. Describe what you changed and why. We review everything.

Good first issues

Look for issues labeled good first issue on GitHub. These are scoped, well-described, and a great way to get familiar with the codebase.
